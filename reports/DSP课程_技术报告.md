# DSP课程技术报告

## 1. 项目概述

本项目围绕经典 Klatt 共振峰语音合成器开展二次开发，目标是在纯 Python 环境下完成一个适合数字信号处理课程展示的实验系统。系统不仅能够根据共振峰参数生成语音样本，还构建了一个基于 PyQt5 的交互式面板，用于手动拼接音节、试听合成结果、导出 WAV 音频以及导出用于实验报告的 IEEE 风格图像。

从课程目标看，本项目的价值在于将离散时间信号、IIR/FIR 滤波器、共振峰建模、频谱分析、语音合成和图形可视化整合到一个统一的实验平台中，使抽象的 DSP 概念能够通过可听、可视、可交互的方式被观察和验证。

## 2. 系统总体架构

系统采用“底层合成引擎 + 音节预设层 + 可视化层 + 图形交互层”的四层结构。

1. 底层合成引擎
   负责实现 Klatt 语音合成的核心 DSP 运算，包括声门激励、噪声源、共振峰滤波和反共振处理。该部分对应 `klatt_syn/klatt.py`。

2. 音节预设层
   负责将具体音节组织成统一的数据结构，包括元音、双元音和辅音段预设。该层通过 JSON 文件管理参数，核心接口在 `klatt_syn/syllables.py`，预设数据位于 `presets/syllables.json`。

3. 可视化层
   负责根据音频序列绘制波形图、声谱图、共振峰轨迹图以及音节/声源/F0 时间轴图，形成适合课程报告的 IEEE 风格图像。该部分位于 `klatt_syn/visualization.py`。

4. 图形交互层
   负责构建 PyQt5 面板，实现音节筛选、音节拼接、试听、排序、WAV 导出和图像导出。该部分位于 `app/pyqt_panel.py`。

该架构将参数模型、信号处理、可视化和交互逻辑解耦，便于后续继续扩展音节库、可视化样式和实验分析功能。

## 3. 核心 DSP 模块设计

### 3.1 声门激励模块

系统支持三类激励源：

- `impulsive`：脉冲型声门激励，适合元音和有声段
- `natural`：更平滑的自然声门模型
- `noise`：纯噪声激励，适合 /h/、/s/、/sh/、/f/ 等无声辅音

这种设计体现了语音源-滤波器模型的思想，即先产生激励，再由声道滤波器改变其频谱结构。

### 3.2 滤波器模块

底层实现中包含以下关键滤波器：

- 一阶低通 IIR 滤波器
- 二阶共振器 Resonator
- 二阶反共振器 AntiResonator
- 一阶差分滤波器

从 DSP 角度看，这些模块分别对应频谱倾斜、共振峰增强、鼻腔反共振建模以及高频能量补偿等功能。尤其是共振器模块直接决定了合成语音的主要频率峰位置，是语音音色形成的核心。

### 3.3 级联/并联合成结构

Klatt 模型的一个典型特征是同时支持 cascade 和 parallel 两条支路：

- cascade 支路更适合描述稳定的元音声道
- parallel 支路更适合加入摩擦噪声、高频旁路等成分

本项目在当前预设中以 cascade 为主，以便保证元音的稳定性；但在系统架构上保留 parallel 参数接口，为后续更复杂的擦音和塞擦音实验提供基础。

## 4. 音节预设与参数组织

### 4.1 预设管理方式

项目没有把所有音节硬编码在 GUI 中，而是采用 JSON 外部配置文件统一管理。这样做的优势包括：

- 新增音节无需修改界面逻辑
- 便于课程实验中对参数做版本对比
- 可将音节、元音、辅音视为一组可复用的实验样本库

### 4.2 当前预设覆盖范围

当前系统已经支持以下 17 个预设：

- 元音：`a`, `i`, `u`, `e`, `o`, `ae`
- 双元音：`ai`, `ei`, `ou`
- 带辅音起始的完整音节：`ha`, `hi`
- 独立辅音或辅音段：`h`, `s`, `sh`, `f`, `m`, `n`

其中，`hi` 被设计为 `/h/ + /ai/` 两段结构，`ai`、`ei`、`ou` 则通过多帧参数插值实现共振峰的时间过渡。这种设计能够较好体现时变语音信号的本质。

## 5. 可视化与报告输出设计

为了适应 DSP 课程报告的展示需求，本项目将四类关键图像直接集成到面板中，并支持 300 DPI 导出：

- Waveform：展示语音信号时域形态
- Spectrogram：展示时间-频率能量分布
- Formant Trajectories：展示 F1/F2/F3 的时间变化
- Sequence Timeline / F0：展示音节结构、声源类型和基频轨迹

采用这种设计的原因是，课程报告通常需要同时说明：

- 信号在时域上的连续性
- 频域上的共振峰结构
- 双元音或辅音-元音过渡中的参数变化
- 拼接策略是否平滑

因此，音频输出和图像输出必须同步存在，才能形成完整的实验链路。

## 6. GUI 交互设计

PyQt5 面板采用“左侧控制区 + 右侧图像区”的布局：

- 左侧为音节选择、序列管理、试听和导出控制
- 右侧为四张独立报告图

这种设计遵循以下原则：

1. 合成控制与分析结果并列展示
   用户可以一边调整音节序列，一边即时观察波形和频谱变化。

2. 图像直接服务于实验报告
   面板中的图像不是装饰，而是为了直接导出并写入课程报告。

3. 风格统一
   所有图像统一采用 300 DPI、Times New Roman、小字号排版，以贴近 IEEE 论文风格和课程正式报告风格。

## 7. 测试与验证

项目目前已建立基础自动化测试，主要覆盖以下几个方面：

- 预设 JSON 能否正确加载
- 非法 glottal source type 是否能被拒绝
- 缺失 frames、空 presets 等错误输入是否能触发异常
- `hi` 单音节能否成功生成非空波形
- `hi + hi` 序列拼接后长度是否合理
- IEEE 图像导出是否能生成 4 个独立 PNG 文件
- GUI 内部序列管理操作是否正确

这些测试保证了系统在课程展示和报告撰写阶段具有基本稳定性。

## 8. 技术特点与课程价值

从 DSP 课程的角度，本项目具有以下技术特点：

- 将经典语音合成理论转化为可执行系统
- 将 IIR/FIR 滤波器与语音建模结合
- 将参数化语音生成与图像分析打通
- 通过 GUI 降低实验操作门槛
- 支持以图、表、音频三种形式输出实验结果

与直接调用黑盒 TTS 模型相比，本项目更适合作为 DSP 课程实验，因为系统内部结构透明、参数含义明确，便于教学、分析和讨论。

## 9. 当前局限

尽管系统已经具备较完整的课程展示能力，但仍存在以下限制：

- 音节参数主要依赖手工设计，缺少自动标定过程
- 自然度有限，当前目标更偏向可解释性与可辨识性
- 尚未实现完整文本到语音流程
- 并联支路在当前预设中使用较少，复杂辅音建模仍可继续扩展
- 图像分析尚未加入自动共振峰估计或客观语音质量评价指标

## 10. 后续扩展建议

为进一步提升课程项目质量，后续可以从以下方向扩展：

- 增加更多英语元音和辅音组合音节
- 增加完整音节模板，如 `sa`, `shi`, `ma`, `na`, `fo`
- 增加参数编辑面板，支持手工调节 F1/F2/F3
- 加入批量导出模式，自动生成音频与图像素材
- 引入客观评价指标，如能量、谱重心、零交叉率等
- 增加对比实验，将不同参数设置下的输出进行成组分析

## 11. 结论

本项目成功构建了一个面向 DSP 课程实验的参数化语音合成平台。系统以 Klatt 共振峰语音合成为核心，通过 Python 和 PyQt5 实现了从音节参数建模、语音生成、交互式拼接到 IEEE 风格图像导出的完整流程。

从工程实现角度看，项目结构清晰、模块职责明确、测试覆盖基础功能；从课程实验角度看，项目能够将语音合成中的时域、频域、滤波器和参数控制概念具体化、可视化，是一个兼具实现价值和教学价值的 DSP 应用案例。
